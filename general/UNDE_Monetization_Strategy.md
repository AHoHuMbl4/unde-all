# **1\. Философия монетизации**

## **1.1. Главный принцип**

*"UNDE растёт за счёт бизнеса. Бизнес приводит нам клиентов, чтобы получить право продвигаться."*

Это инверсия классической модели. Не мы платим за привлечение пользователей — бизнес платит нам трафиком за доступ к экосистеме.

| Классическая модель | Модель UNDE |
| :---- | :---- |
| Платформа платит за трафик | Бизнес приводит трафик платформе |
| Бизнес платит за размещение | Бизнес зарабатывает право быть видимым |
| Транзакционные отношения | Экосистемная зависимость |
| Конкуренты копируют функции | Конкуренты не могут скопировать зависимость |

## **1.2. Моральный контракт**

| UNDE — на стороне пользователя. Всегда. Любая монетизация подчиняется моральному контракту: мы не продаём внимание пользователя. Мы усиливаем релевантность. Если бизнес-модель противоречит интересам пользователя — мы выбираем пользователя. |
| :---- |

UNDE — часть ритуала пользователя. Не инструмент, который открывают по необходимости, а привычка, которая встроена в жизнь:

* «Иду в ТЦ → открываю UNDE» — автоматизм  
* «Вижу вещь в Instagram → открываю в UNDE» — рефлекс  
* «Не знаю что надеть → спрашиваю UNDE» — доверие

Монетизация не должна разрушать этот ритуал. Пользователь никогда не должен почувствовать, что UNDE «продал» его внимание.

## **1.3. Принцип релевантности**

| Пользователь всегда получает ценность без рекламы Продвижение товара происходит ТОЛЬКО среди релевантных альтернатив (по стилю, бюджету, размеру). Если товар не подходит пользователю — он никогда не появится в рекомендациях, сколько бы Credits ни потратил бизнес. Мы не продаём внимание — мы усиливаем релевантность. |
| :---- |

Буст работает только среди «равных»:

* AI подбирает 10 релевантных товаров для пользователя  
* Бизнес с Credits может поднять свой товар выше в этих 10  
* Но нерелевантный товар НИКОГДА не попадёт в эту десятку

# **2\. Модель Earn-to-Spend**

## **2.1. Ключевая механика**

*"Чтобы купить рекламу — сначала вложись в рост экосистемы."*

Формула доступа к покупке Credits:

**Максимум покупки \= Заработанные Credits × K**

Где K — коэффициент, зависящий от уровня вовлечённости бизнеса.

| Заработано Credits | Коэффициент K | Можно докупить | Итого |
| :---- | :---- | :---- | :---- |
| 0 | — | 0 | 0 |
| 1,000 | 0.3 | 300 | 1,300 |
| 5,000 | 0.5 | 2,500 | 7,500 |
| 20,000 | 0.7 | 14,000 | 34,000 |
| 50,000+ | 1.0 | 50,000 | 100,000 |

## **2.2. Как бизнес зарабатывает Credits**

Credits начисляются только за КАЧЕСТВЕННЫХ пользователей — не за установки, а за активации и retention:

| Действие пользователя | Credits | Почему это ценно |
| :---- | :---- | :---- |
| Установил app по ссылке бизнеса | \+10 | Минимальный сигнал интереса |
| Прошёл онбординг \+ первая примерка | \+100 | Activated user |
| Вернулся на D7 | \+50 | Формируется привычка |
| Вернулся на D30 | \+150 | Лояльный пользователь |
| Пошерил образ | \+30 | Виральность |
| Совершил покупку | \+200 | Конверсия \= главная ценность |

| Антифрод Сама по себе установка даёт ничтожно мало Credits (\+10). Реальные Credits начисляются только за activated и retained пользователей — тех, кто прошёл онбординг, вернулся на D7/D30 или совершил покупку. Фейковые установки, которые не конвертируются в активацию, практически не влияют на баланс. Это делает накрутки экономически бессмысленными. |
| :---- |

# **3\. Сгорание Credits**

## **3.1. Механика expiration**

*"Credits — это не накопления. Это показатель активного участия в экосистеме."*

Правила сгорания:

* Credits действительны 90 дней с момента последней активности  
* Если бизнес не привёл ни одного пользователя за 90 дней — начинается decay  
* Decay rate: 10% от баланса в неделю  
* Любая активность (привлечение, обновление каталога) обнуляет таймер

## **3.2. Визуализация для бизнеса**

В дашборде бизнес видит:

| Элемент | Что показывает |
| :---- | :---- |
| Баланс Credits | Текущее количество \+ динамика за месяц |
| Таймер сгорания | «67 дней до начала decay» / «Decay активен: \-10%/нед» |
| Последняя активность | Дата \+ тип (привлечение/каталог/лук) |
| Прогноз | «При текущей активности через 30 дней: X Credits» |

| Почему сгорание критично Без expiration крупные игроки накопят Credits и «заморозят» топовые позиции. Сгорание создаёт постоянный стимул к активности и даёт шанс новичкам. |
| :---- |

# **4\. Что бизнес получает за Credits**

UNDE продаёт бизнесу три вещи, которых в офлайне не существует: **атрибуцию** (цепочка «рекомендация → визит → покупка»), **данные о намерениях** (что конкретный человек ищет прямо сейчас в ТЦ) и **конкурентную разведку** (откуда пришёл клиент и куда ушёл). Каждая из трёх — отдельный рынок на миллиарды в онлайне, и ноль предложений в офлайне.

## **4.1. Три уровня ценности**

| Уровень | Что получает бизнес | Модель |
| :---- | :---- | :---- |
| **Трафик** | Атрибутированные визиты + контекстные рекомендации (UNDE сам решает, когда напомнить о релевантном рядом). **UNDE (цель): $3–10 за verified-visit** (параметр монетизации). Для сравнения: в drive‑to‑store кейсах встречается **$0.25 CPSV** (cost per store visit) и примеры **~$3.85 CPSV / ~$20 CPIV** (cost per incremental visit) — разброс зависит от методологии и lift‑расчёта. Store Visits в Google Ads — **моделируемая оценка**, а не верифицированная продажа. [S7][S8][S1] | Аукцион Credits |
| **Аналитика** | Эксклюзивные данные: откуда пришёл клиент, куда ушёл, что примерял vs купил, конверсия по часам. Для ориентира по стоимости внешней footfall‑аналитики: в открытых муниципальных документах встречаются контракты Placer.ai/Placer Labs на **$10k–$12k/год** (пример Smithville) и **$31k/год** (пример City of Bloomington) — без учёта медиа‑бюджета. [S5][S6] | Подписка (фаза 3) |
| **Продвижение** | Приоритет в AI-рекомендациях среди релевантных товаров. Аналог логики retail media networks: по данным Nielsen (со ссылкой на eMarketer), **retail media spending в США достигнет ~$60B в 2025** и **~$100B к 2028**. [S3] | Аукцион Credits |

## **4.2. Аукционная механика (единая для всех форматов)**

Все форматы продвижения работают по одному принципу:

* AI подбирает релевантные товары для пользователя (по стилю, бюджету, размеру)
* Бизнес вкладывает X Credits → его товар поднимается выше в выдаче
* Конкурент вкладывает столько же → он на первом месте, предыдущий на втором
* Нерелевантный товар НИКОГДА не попадёт в выдачу — моральный контракт

Нет фиксированных ставок — рынок определяет цену. Это работает для буста в рекомендациях, «Образа дня» и контекстных рекомендаций при нахождении рядом с магазином.

**Контекстные рекомендации (при нахождении рядом с магазином):**

Бизнес НЕ может отправить уведомление конкретному пользователю. UNDE (AI) сам решает, релевантно ли предложение магазина для этого клиента. Бизнес через аукцион Credits влияет на приоритет среди релевантных — но триггер всегда на стороне UNDE. Бизнес видит только агрегат: «14 контекстных рекомендаций сегодня», без персональных данных.

## **4.3. Воронка списания Credits (пример, не утверждённая механика)**

| Событие | Credits (пример) |
| :---- | :---- |
| Показ в рекомендациях | 2 |
| Клик «Где купить» | 5 |
| Подтверждённый визит (нахождение рядом с магазином) | 20 |
| **Полная воронка** | **27** |

## **4.4. Модель атрибуции (послойная)**

Атрибуция наращивается по слоям — от базовой (без участия магазина) до полной:

| Слой | Метод | Зависимость от магазина |
| :---- | :---- | :---- |
| Цифровой интент | Показ → клик «Где купить» | Нет |
| Нахождение рядом | Клиент в зоне магазина >3 мин | Нет |
| Самоотчёт | «Я купил!» → бонус в UNDE | Нет |
| Фото чека | Распознавание магазина, суммы, даты | Нет |
| QR / промокод | Стикер в магазине → скан → подтверждение | Нужен договор |
| Интеграция с кассой | Прямая связь с кассовой системой | Глубокое партнёрство |

**Для MVP:** слои 1–3. **Для масштаба:** + фото чека. **Для крупных партнёров:** QR / интеграция с кассой.

Ключевая механика: клиент сканирует чек не «для магазина», а для прокачки своего аватара UNDE. Двойная ценность: для клиента — аватар прокачался, вещь в цифровом гардеробе; для бизнеса (невидимо) — 100% атрибуция покупки.

## **4.5. Аналитика в дашборде**

UNDE знает полный путь пользователя по ТЦ за счёт разработанной уникальной системы навигации внутри помещений без дополнительного оборудования — даже без интеграции с другими бизнесами.

| Данные | Пример инсайта | Ценность |
| :---- | :---- | :---- |
| Откуда пришёл клиент | «34% — сразу к вам, 28% — после H\&M» | Конкурентная разведка |
| Куда ушёл после | «41% — купил и ушёл, 23% — фудкорт» | Потерянные клиенты |
| Что примерял vs купил | «Примеряют синее, покупают чёрное» | Оптимизация ассортимента |
| Время визита | «Конверсия в 14:00–16:00 на 2.3x выше» | Оптимизация расписания |
| Паттерны до покупки | «Клиенты после кофе конвертятся лучше» | Партнёрства |

## **4.6. Сравнение с текущими расходами ритейлера**

Единого «среднего» бюджета для premium-fashion нет: он резко зависит от страны, площади, арендной ставки, payroll и медиамикса. Поэтому вместо неподтверждённого диапазона расходов фиксируем сравнимые **проверяемые** бенчмарки:

- **Footfall / location intelligence analytics:** публичные документы по Placer.ai показывают примеры годовых подписок порядка **$10k–$31k/год** (~$0.8k–$2.6k/мес). [S5][S6]
- **Drive-to-store медиабюджеты:** в опубликованных кейсах cost per store visit может быть **$0.25** (CPSV) и выше; при расчёте инкрементальности встречается порядок **~$20 за incremental visit** (CPIV). [^7][^8]
- **Важно про атрибуцию:** «Store Visits» в Google Ads — моделируемая оценка и может флуктуировать/дополняться моделью с задержкой. [S1][S2]

UNDE (предложение в этом документе): **$3,000–5,000/мес** как «объединённый пакет» (in-app demand + verified attribution на уровне пользователя/чека + analytics + CRM loops). Финальная цена и SLA по атрибуции зависят от интеграции (например, чек/QR vs только визиты).


# **5\. Индекс UNDE**

## **5.1. Что это**

*"Индекс UNDE — единый показатель вовлечённости бизнеса в экосистему."*

Индекс рассчитывается для каждого бизнеса в каждом конкретном молле (не глобально).

| Амбиция: сделать Индекс UNDE стандартом офлайн-ритейла Как NPS стал стандартом для измерения лояльности, Индекс UNDE должен стать стандартом для измерения эффективности бизнеса в офлайн-ритейле. «Какой у вас Индекс UNDE?» — вопрос, который будут задавать инвесторы, арендодатели, партнёры. |
| :---- |

## **5.2. Компоненты индекса (пример, не утверждённая механика)**

Возможные компоненты и их примерные веса (будут уточняться по мере тестирования):

| Компонент | Примерный вес | Что измеряет |
| :---- | :---- | :---- |
| Привлечение | ~40% | Сколько activated пользователей привёл |
| Retention | ~25% | D30 retention привлечённых |
| Каталог | ~15% | Актуальность и полнота данных |
| Конверсия | ~20% | % рекомендаций → покупка |

## **5.4. Индекс UNDE как отраслевой стандарт**

Долгосрочная цель — сделать Индекс UNDE метрикой, которую используют:

* Инвесторы — для оценки эффективности ритейл-бизнеса  
* Арендодатели ТЦ — для определения арендной ставки  
* Бренды — для выбора партнёров и локаций  
* Аналитики рынка — для отраслевых отчётов

# **6\. Метрики успеха**

## **6.1. Единица роста: Mall Seed**

*"«Всего скачало X раз» — vanity metric. Единица роста UNDE — Mall Seed."*

| Что такое Mall Seed Mall Seed — минимальная жизнеспособная сеть в одном ТЦ. Это точка, после которой экосистема начинает расти органически. Mall Seed \= 500+ MAU \+ 10+ активных бизнесов \+ k-factor \> 1.0 в рамках одного молла. |
| :---- |

Все метрики считаются для каждого молла отдельно:

| Метрика | Не так | А так |
| :---- | :---- | :---- |
| Пользователи | «100K установок» | «2,400 MAU в Aport Mall» |
| Retention | «D30 \= 45%» | «D30 в Aport Mall \= 52%» |
| Конверсия | «8% покупают» | «11% покупают в Aport Mall» |
| Credits | «Всего 1M Credits» | «Zara: 12,450 Credits в Aport Mall» |
| Бизнесы | «50 партнёров» | «12 активных бизнесов в Aport Mall» |

## **6.2. Почему Mall Seed, а не глобальные метрики**

Глобальные метрики скрывают реальную картину:

* «100K установок» может означать 1,000 юзеров в 100 моллах \= нет критической массы нигде  
* Network effects работают локально: пользователь ценит UNDE в СВОЁМ молле  
* Бизнес платит за трафик в СВОЁМ молле, не глобально

# **7\. Расширение категорий**

## **7.1. Стратегия: Fashion → Весь молл**

*"Fashion — точка входа. Цель — стать операционной системой всего ТЦ."*

Fashion выбран как первая категория потому что:

* В enclosed malls (пример: США) apparel & accessories stores занимают **57.8%** всей non-anchor площади (ICSC) — fashion почти всегда крупнейшая категория, что делает её сильной точкой входа для cold start. [S4]
* Высокая эмоциональная вовлечённость (люди любят делиться образами)  
* Виральный контент (фото образов \= UGC)  
* Высокий средний чек и маржинальность  
* Чёткий JTBD: «Что надеть?»

## **7.2. Roadmap расширения**

| Год | Категории | AI-персона | Ценность данных |
| :---- | :---- | :---- | :---- |
| 1 | Fashion | AI-стилист | Что примерял, купил, стиль |
| 2 | \+ Food | AI-гастроном | Полный путь: шопинг → еда |
| 3 | \+ Entertainment, Services | AI-консьерж | Полная картина дня в ТЦ |
| 4+ | Все категории ТЦ | AI-агент молла | UNDE \= ОС молла |

Расширение категорий неограничено. Конечная цель — охватить весь lifestyle пользователя и в каждом контексте быть лучшим. Там, где собственная экспертиза недостаточна, UNDE подключает лучшие специализированные решения по API, выступая как AI-оркестратор, который определяет потребность пользователя и маршрутизирует её к оптимальному партнёрскому решению.

## **7.3. Ценность полного пути**

Когда UNDE знает весь путь пользователя по ТЦ:

| Для кого | Инсайт | Действие |
| :---- | :---- | :---- |

| Paul | «34% клиентов приходят из fashion» | Промо «кофе после шопинга» |

| Управляющая ТЦ | «Мёртвая зона у входа B» | Пересмотр размещения |

| Конечная цель UNDE \= единый интерфейс между человеком и всем торговым центром. Не «приложение для шопинга», а операционная система офлайн-ритейла. В дальнейшем — расширение до суперэксперта в гео вокруг пользователя: рестораны, услуги, развлечения, любые точки интереса в радиусе доступности. |

# **8\. Дорожная карта монетизации**

## **8.1. Фазы**

| Фаза | Период | Модель | Фокус |
| :---- | :---- | :---- | :---- |
| Earn-only | 0–18 мес | Только заработок Credits | Максимум бизнесов и юзеров |
| Earn-to-Buy | 18–36 мес | Докупка с ограничениями | Выручка \+ рост |
| Full Ecosystem | 36+ мес | Подписки \+ комиссии | Масштабирование прибыли |

## **8.2. Фаза 1: Earn-only (0–18 мес)**

* Нельзя купить Credits — только заработать  
* API интеграция каталога — бесплатно навсегда  
* Цель: максимум Mall Seeds, создание зависимости

## **8.3. Фаза 2: Earn-to-Buy (18–36 мес)**

* Можно докупить Credits, но только % от заработанного  
* Прогрессивные коэффициенты K (0.3 → 0.5 → 0.7 → 1.0)  
* Выручка \= Credits × K × $0.10

## **8.4. Фаза 3: Full Ecosystem (36+ мес)**

* Подписка на расширенную аналитику: $500–2,000/мес  
* Комиссия с атрибутированных покупок: 2–5%

## **10.2. Что бизнес теряет при уходе**

2. Накопленные Credits — репутация не переносится  
3. Индекс UNDE — позиция в рейтинге молла  
4. Аналитику пути — «откуда пришёл / куда ушёл»  
5. Статус партнёра — маркетинговый актив

# **11\. Принципы (не нарушать)**

| Принцип | Что это значит | Почему важно |
| :---- | :---- | :---- |
| Только свой бренд | Никакого white-label. Развиваем только UNDE | Бренд \= главный актив |
| API всегда бесплатно | Интеграция каталога — $0 | Максимум данных \= лучше рекомендации |
| Метрики по моллам | Всё считаем в разрезе конкретного ТЦ | Реальная картина, не vanity |
| Mall Seed \= единица роста | Фокус на критической массе в одном молле | Network effects локальны |
| Релевантность \> деньги | Буст только среди подходящих товаров | Доверие пользователя священно |
| Моральный контракт | UNDE всегда на стороне пользователя | Ритуал важнее выручки |
| Ценность без рекламы | Пользователь получает пользу всегда | Реклама — усиление, не замена |
| Earn-to-Spend | Сначала вложись — потом покупай | Экосистема растёт за счёт бизнеса |
| Fashion → Весь молл | Категории расширяются постепенно | Глубина важнее ширины |
| Индекс UNDE \= стандарт | Амбиция — отраслевая метрика | Определяем правила игры |

| Итог: почему это работает UNDE становится инфраструктурой спроса. Бизнес не может уйти, потому что: (1) мы — единственный источник данных о пути клиента в офлайне, (2) накопленные Credits и Индекс не переносятся, (3) пользователи привыкли искать через нас. Без UNDE хуже. С UNDE — конкурентное преимущество. |

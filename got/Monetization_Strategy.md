**UNDE**

Стратегия монетизации и роста

Внутренний документ | Февраль 2026 | v3.4 (draft)

> **О цифрах в документе.** Все параметры Credits, коэффициенты K, пороги Local Seed и примерные цены — это **продуктовые гипотезы**, подлежащие валидации в пилоте. Внешние ориентиры (retail media, аналитика посещаемости, стоимость визита) отмечены ссылками и проверены по публичным источникам. # 1. Философия монетизации

## 1.1. Главный принцип

**UNDE растёт за счёт бизнеса. Бизнес приводит нам пользователей, чтобы получить право продвигаться.**

Это инверсия классической модели. Не мы платим за привлечение — бизнес платит нам трафиком за доступ к экосистеме.

| Классическая модель | Модель UNDE |
| :---- | :---- |
| Платформа платит за трафик | Бизнес приводит трафик платформе |
| Бизнес платит за размещение | Бизнес зарабатывает право быть видимым |
| Транзакционные отношения | Экосистемная зависимость |
| Конкуренты копируют функции | Конкуренты не могут скопировать зависимость |

## 1.2. Моральный контракт

UNDE — на стороне пользователя. Всегда. Любая монетизация подчиняется моральному контракту:

- Мы не продаём внимание пользователя. Мы усиливаем релевантность.
- Мы не продаём персональные данные. Никому. Никогда.
- Мы не берём комиссию с транзакций пользователя. UNDE — не маркетплейс.
- Если решение о монетизации противоречит интересам пользователя — мы выбираем пользователя.

Каждое решение о монетизации проверяется вопросом: **«Это укрепляет или разрушает ритуал?»** Если разрушает — не делаем, какой бы ни была выручка.

## 1.3. Принцип релевантности (и почему это не манипуляция)

Пользователь всегда получает ценность без рекламы. Продвижение происходит ТОЛЬКО среди релевантных альтернатив (по стилю, бюджету, размеру, контексту):

- AI подбирает N релевантных вариантов для пользователя
- Provider с Credits может повысить приоритет своего варианта среди этих N
- Но нерелевантный вариант НИКОГДА не попадёт в выдачу, сколько бы Credits ни потратил Provider

**Ответ на главный вопрос: «Если Provider платит за приоритет — это не реклама?»**

Нет. И вот почему. Когда человек спрашивает друга «Какие белые кроссовки купить?» — друг выбирает из тех, которые он считает подходящими. Если два варианта одинаково хороши — друг порекомендует тот, о котором знает больше, или тот, с которым у него был лучший опыт. Это не манипуляция — это приоритизация среди равных.

UNDE работает так же. AI сначала отсекает всё нерелевантное (по стилю, бюджету, размеру, доступности). Provider через Credits может получить приоритет только внутри уже отфильтрованного пула. Пользователь в любом случае получает варианты, которые ему подходят. Разница — в порядке, не в качестве.

Ключевой тест: **если убрать все Credits из системы — рекомендации останутся полезными.** Credits не создают рекомендации, а расставляют приоритеты среди уже хороших вариантов. Если этот тест перестанет проходить — модель сломана.

**Архитектурная гарантия: Organic Floor.**

Этот тест не только теоретический. UNDE встраивает его в работу системы:

- X из каждых Y рекомендаций показываются в чисто органическом порядке, без влияния Credits (гипотеза для пилота: 3 из 10)
- Provider не знает, какие именно показы органические
- Пользователь не видит разницы (все варианты релевантны в обоих случаях)

Что это даёт:

- **Пользователю:** паттерн «всегда Zara наверху» физически не возникает. Даже если один Provider единственный платит за приоритет — в значительной доле показов порядок определяется только AI-релевантностью
- **Provider'у:** честный сигнал. Credits дают приоритет, но не монополию. Если товар реально лучше подходит — он и без Credits появляется в органических показах
- **UNDE:** встроенный A/B-тест. Органические показы = контрольная группа. Можно измерять: «повышают ли Credits конверсию или просто перераспределяют её?» Данные для калибровки всей системы
- **Моральному контракту:** тест «убери Credits — всё работает» проходит буквально в каждой Y-й рекомендации, а не как мысленный эксперимент

## 1.4. Что мы НЕ делаем (и почему)

| Чего не делаем | Почему |
| :---- | :---- |
| Комиссия с транзакций | UNDE — не маркетплейс. Комиссия создаёт конфликт интересов: мы начинаем зарабатывать на покупке, а не на доверии |
| Продажа персональных данных | Убивает доверие мгновенно. Один заголовок «AI-друг продаёт данные» — и защитный ров отношений обнуляется |
| «UNDE Ads» / баннерная реклама | Превращает слой доверия в рекламную сеть. Пользователь перестаёт верить рекомендациям |
| Скоринг пользователей | «AI-друг оценивает тебя для банков» — репутационная бомба |
| Продажа платформы под чужим брендом | Развиваем только UNDE. Бренд = главный актив |

# 2. Что продаёт UNDE

## 2.1. Не реклама — доступ к намерению в моменте

UNDE продаёт Provider'ам не «показы» и не «клики», а доступ к человеку, который:

- уже сформулировал потребность
- находится в зоне доступности (от текущего ТЦ до города — см. ниже)
- видит варианты, которые реально доступны сейчас

Это **Intent Performance** (оплата за результат из релевантного пула намерения) — не за охват аудитории.

**Зона доступности — расширяющийся поиск.** UNDE ищет варианты, начиная с ближайших и расширяя радиус:

1. Текущий ТЦ (приоритет — можно дойти прямо сейчас)
2. Ближайшие ТЦ / магазины в городе (с пометкой расстояния и времени)
3. Онлайн-наличие (если офлайн-вариантов нет)

Если белых кроссовок нет в этом молле — UNDE покажет, где они есть в городе, с оговоркой «это в другом месте, 20 мин на машине». Пользователь всегда получает ответ. Provider'ы за пределами текущего ТЦ тоже участвуют в Intent Performance — но с меньшим приоритетом при прочих равных (ближайший вариант ценнее для пользователя).

**Ближайшая аналогия: Google Search.** Google не берёт комиссию с покупок. Google продаёт доступ к намерению — момент, когда человек уже решил что-то найти. Эта модель сделала Google одной из крупнейших компаний мира. UNDE делает то же самое, но для офлайна: продаёт Provider'ам доступ к намерению человека, который уже решил что-то купить/сделать и находится в зоне доступности. С одним ключевым отличием: моральный контракт гарантирует, что доступ возможен только к релевантным намерениям. Нерелевантный Provider не получит доступ ни за какие деньги.

**Масштаб возможности:** UNDE становится стандартным интерфейсом выбора в офлайне. Не маркетплейс, не рекламная сеть — а инфраструктура, через которую проходит решение «что/где/когда» в физическом мире. Каждое такое решение — точка монетизации.

## 2.2. Три вещи, которых нет в офлайне

UNDE закрывает три разрыва, каждый из которых — отдельный рынок в онлайне и ноль предложений в офлайне:

| Разрыв | Что это | Аналог в онлайне | Почему этого нет в офлайне |
| :---- | :---- | :---- | :---- |
| **Атрибуция** | Цепочка «рекомендация → визит → покупка» | Google Analytics, UTM | Store Visits в Google Ads — modeled estimate, не верификация [S1] |
| **Intent data** | Что конкретный человек ищет прямо сейчас | Search intent (Google) | В офлайне человек просто ходит — его намерение невидимо |
| **Конкурентная разведка** | Откуда пришёл клиент и куда ушёл | Retargeting, cohort analysis | Магазин видит только тех, кто зашёл |

## 2.3. Универсальные определения

Архитектура монетизации построена на трёх универсальных абстракциях, которые работают для любой вертикали:

**Provider** — любой бизнес, подключённый к UNDE: магазин, салон, клиника, мастер, автосервис, ресторан. На старте: fashion-ритейлеры в ТЦ.

**Real-Time State Feed** — данные о текущем состоянии Provider'а: наличие товара, цены, промо, свободные слоты, время ожидания, загруженность. На старте: каталог одежды (наличие, размеры, цены).

**Verified Action** — универсальная единица подтверждённого результата:

| Вертикаль | Примеры Verified Actions |
| :---- | :---- |
| Retail (старт) | Визит к магазину, нахождение у кассы, покупка (чек) |
| Services (будущее) | Бронирование, факт оказания услуги, оплата |
| Booking (будущее) | Подтверждённый слот, визит |

Единый движок монетизации: Credits зарабатываются и тратятся на Verified Actions одинаково для Zara, парикмахерской и автосервиса.

# 3. Credits Economy

## 3.1. Механика Earn-to-Spend

*«Чтобы получить доступ к спросу — сначала вложись в рост экосистемы.»*

Формула: **Максимум покупки = Заработанные Credits × K**

Где K — коэффициент, зависящий от уровня вовлечённости Provider'а.

| Заработано Credits | Коэффициент K | Можно докупить | Итого |
| :---- | :---- | :---- | :---- |
| 0 | — | 0 | 0 |
| 1,000 | 0.3 | 300 | 1,300 |
| 5,000 | 0.5 | 2,500 | 7,500 |
| 20,000 | 0.7 | 14,000 | 34,000 |
| 50,000+ | 1.0 | 50,000 | 100,000 |

Ключевое: нулевой баланс = нулевой доступ. Нельзя прийти с деньгами и «купить рекламу» — сначала докажи, что ты полезен экосистеме.

## 3.2. Как Provider зарабатывает Credits

Credits начисляются только за **качественных** пользователей — не за установки, а за activated и retained:

| Действие пользователя | Credits | Почему это ценно |
| :---- | :---- | :---- |
| Установил app по ссылке Provider'а | +10 | Минимальный сигнал интереса |
| Прошёл онбординг + первая примерка | +100 | Activated user |
| Вернулся на D7 | +50 | Формируется привычка |
| Вернулся на D30 | +150 | Лояльный пользователь |
| Пошерил образ | +30 | Виральность |
| Совершил Verified Action (покупка) | +200 | Конверсия = главная ценность |

**Антифрод:** установка даёт ничтожно мало (+10). Реальные Credits — за activation и retention. Фейковые установки без конверсии в активность экономически бессмысленны.

## 3.3. На что тратятся Credits

Все форматы работают через единый механизм Intent Performance:

- AI подбирает релевантные варианты для пользователя (по стилю, бюджету, размеру)
- Provider вкладывает X Credits → его вариант получает приоритет в выдаче среди релевантных
- Конкурент вкладывает больше → он получает более высокий приоритет
- Нерелевантный вариант НИКОГДА не попадёт в выдачу — моральный контракт

Нет фиксированных ставок — рынок определяет цену. Это работает для:

- Приоритета в рекомендациях (основной формат)
- Контекстных рекомендаций при нахождении в зоне Provider'а (UNDE решает, релевантно ли — Provider влияет только на приоритет среди равных)
- «Образа дня» и других промо-форматов

**Organic Floor (см. §1.3):** X из каждых Y показов — чисто органические, без влияния Credits. Provider не знает, какие. Это гарантирует, что Credits дают приоритет, но не монополию, и обеспечивает постоянную калибровку системы.

**Важно:** Provider НЕ может отправить уведомление конкретному пользователю. UNDE (AI) сам решает, когда и что показать. Provider через Credits влияет на приоритет среди релевантных — но триггер всегда на стороне UNDE. Provider видит только агрегат: «14 контекстных рекомендаций сегодня», без персональных данных.

## 3.4. Expiration и decay

*«Credits — не накопления. Это показатель активного участия в экосистеме.»*

- Credits действительны 90 дней с момента последней активности
- Если Provider не привёл ни одного пользователя за 90 дней — начинается decay
- Decay rate: 10% от баланса в неделю
- Любая активность (привлечение, обновление каталога) обнуляет таймер

**Почему сгорание критично:** без expiration крупные игроки накопят Credits и «заморозят» топовые позиции. Сгорание создаёт постоянный стимул к активности и даёт шанс новичкам.

## 3.5. Как Provider приводит трафик

Provider получает уникальные deep links на луки/товары:

Клик → открывается app → пользователь видит лук на аватаре → «Примерить на себя?»

| Формат | Где размещает | CTA |
| :---- | :---- | :---- |
| Пост в соцсетях | Instagram, TikTok | «Примерь этот лук в UNDE» |
| QR в магазине | Витрина, примерочная | «Виртуальная примерка» |
| Email / SMS | CRM-рассылки | «Посмотри новую коллекцию на себе» |
| Сайт бренда | Карточка товара | «Открыть в UNDE» |

# 4. UNDE Index

## 4.1. Что это

*«UNDE Index — единый показатель вовлечённости Provider'а в экосистему.»*

Индекс рассчитывается для каждого Provider'а в каждом конкретном Local Seed (не глобально).

**Ключевой принцип: Индекс нельзя купить. Его можно только заслужить.**

Покупать можно инструменты результативности (Credits, улучшение каталога, качество обслуживания). Индекс растёт как *следствие* качественной работы. Если Индекс можно купить — он перестаёт быть стандартом качества и превращается в рекламную метрику.

**Правило протокола: Index = допуск к Intent Performance.**

Минимальный UNDE Index — обязательное условие для траты Credits на приоритет. Provider с Index ниже порога может зарабатывать Credits (привлекать пользователей, загружать каталог), но не может их тратить на Intent Performance.

Что это значит:

- **Двойной фильтр входа:** Earn-to-Spend требует «заработай Credits». Index добавляет «и поддерживай качество». Деньги + плохой продукт = нет доступа к рынку.
- **Рынок честный по умолчанию:** не нужна ручная модерация. Index автоматически отсекает Provider'ов, которые не поддерживают актуальный каталог, не обеспечивают качество обслуживания, не привлекают качественных пользователей.
- **Index = инфраструктура, не отчётность.** Без Index не работает ничего — как кредитный рейтинг для финансовых продуктов. Это делает барьер ухода абсолютным: потерять Index = потерять доступ к рынку UNDE.

## 4.2. Компоненты (гипотеза, будут уточняться)

| Компонент | Примерный вес | Что измеряет |
| :---- | :---- | :---- |
| Привлечение | ~40% | Сколько activated пользователей привёл Provider |
| Retention | ~25% | D30 retention привлечённых пользователей |
| Качество данных | ~15% | Актуальность и полнота Real-Time State Feed |
| Конверсия | ~20% | % рекомендаций → Verified Action |

## 4.3. Лестница верификации (Verification Ladder) — открытая методология

Чтобы Index стал отраслевым стандартом, а не «внутренним скорингом UNDE» — методология подтверждений должна быть прозрачной и верифицируемой.

**Классы подтверждений (от слабых к сильным):**

| Класс | Метод | Надёжность | Доступность |
| :---- | :---- | :---- | :---- |
| C1: Digital intent | Показ → клик «Где купить» | Низкая (intent ≠ action) | Всем с Day 1 |
| C2: Proximity | Нахождение в зоне Provider'а > 3 мин | Средняя | Всем с Day 1 |
| C3: Self-report | «Я купил!» + бонус аватару | Средняя | Всем с Day 1 |
| C4: Receipt scan | Фото чека → распознавание | Высокая | MVP+ |
| C5: QR/promo code | Скан стикера в магазине | Высокая | По договору |
| C6: POS integration | Прямая связь с кассой | Максимальная | Глубокое партнёрство |

Index учитывает *класс* подтверждения: C6 весит больше C1. Provider, инвестирующий в более сильную верификацию, получает более точный (и как правило более высокий) Index.

## 4.4. Три ступени к отраслевому стандарту

| Ступень | Что это | Когда |
| :---- | :---- | :---- |
| **Index v1: внутренний допуск** | Индекс определяет видимость и доступ к промо-инструментам внутри UNDE. Методология опубликована, но аудит внутренний | Фаза 1 |
| **Сравнительные Index-отчёты** | B2B-отчёты для девелоперов ТЦ и брендов: «эффективность ваших магазинов vs категория». Методология открыта для партнёров | Фаза 2 |
| **External legitimacy** | Независимый аудит verification ladder. Партнёрства с аналитиками (аналог: аудит viewability в digital ads). Index принимается арендодателями/инвесторами как стандартная метрика | Фаза 3 |

**Амбиция:** сделать UNDE Index стандартом эффективности офлайн-ритейла. «Какой у вас UNDE Index?» — вопрос, который задают инвесторы, арендодатели, партнёры. Как NPS стал стандартом лояльности.

# 5. Monetization Stack

## 5.0. Архитектура: ядро + ускорители

UNDE монетизируется через три слоя. Но это не три равноценных стрима выручки — это **одно ядро и два ускорителя**:

- **Ядро: Intent Performance** — Provider'ы платят за приоритет среди релевантных вариантов. Это главная машина выручки UNDE, которая масштабируется с каждым новым Local Seed.
- **Ускоритель 1: Analytics** — не продукт ради выручки, а инструмент удержания Provider'а и улучшения его результатов. Лучшая аналитика → Provider лучше работает → больше Verified Actions → сильнее ядро.
- **Ускоритель 2: Enterprise** — масштабирование через операторов пространств. Не отдельный бизнес, а канал дистрибуции и покрытия.

Все три слоя усиливают ядро. Если Analytics и Enterprise исчезнут — UNDE продолжит работать. Если исчезнет Intent Performance — модели нет.

## 5.1. Ядро: Intent Performance

**Что:** приоритет в AI-рекомендациях среди релевантных вариантов.

**Механика:** Provider тратит Credits → его товар/услуга получает приоритет в выдаче среди тех, что уже прошли фильтр релевантности.

**Ключевой дифференциатор:** не по аудитории, а по намерению + контексту + доступности. Provider покупает не «показ 1000 людей», а «приоритет, когда человек ищет именно это и находится в зоне доступности». Это модель оплаты за результат по намерению, адаптированная для офлайна.

**Модель выручки:** Credits × K × $0.10, где K — коэффициент Earn-to-Spend (от 0.3 до 1.0, зависит от объёма заработанных Credits — см. §3.1). Гипотеза для Фазы 2.

**Рыночный контекст:** retail media spending в США — ~$58.8B в 2025 [S3], глобально — ~$169.6B [S10]. Но это онлайн. В офлайне этот рынок пуст. UNDE создаёт его.

## 5.2. Ускоритель 1: Analytics & Attribution

**Что:** агрегированные данные о поведении пользователей в привязке к локации — не отдельный data-продукт, а инструмент, который повышает ROI Provider'а на Credits и усиливает ядро Intent Performance.

Технология indoor-позиционирования UNDE определяет нахождение пользователя в зоне конкретных точек в ТЦ, используя только стандартные датчики смартфона. Все данные о перемещениях обрабатываются в агрегированном виде — Provider'ы видят когорты и паттерны, не индивидуальные траектории.

| Данные | Пример инсайта | Как усиливает ядро |
| :---- | :---- | :---- |
| Потоки посетителей | «34% когорты — сразу к вам, 28% — после H&M» | Provider оптимизирует ассортимент → растёт конверсия → больше Verified Actions |
| Потерянные клиенты | «41% когорты купили и ушли, 23% — фудкорт» | Понимание потерянных клиентов → лучшее обслуживание → выше Index |
| Примерка vs покупка | «Примеряют синее, покупают чёрное» | Точнее каталог → точнее рекомендации → сильнее маховик доверия |
| Время визита | «Конверсия в 14:00–16:00 на 2.3x выше» | Оптимизация расписания → выше ROI на Credits |
| Паттерны до покупки | «Когорта после кофе конвертятся лучше» | Партнёрства между Provider'ами → рост экосистемы |

**Privacy:** только агрегированные данные и когорты. Никаких персональных траекторий. UNDE не продаёт данные о перемещениях конкретных пользователей — ни Provider'ам, ни третьим сторонам.

**Модель:** базовая аналитика входит в пакет Intent Performance — каждый Provider, тратящий Credits, автоматически видит результаты (конверсия, окупаемость, сравнение с органическими показами). Расширенная аналитика (конкурентная разведка, глубокие когортные отчёты, оптимизация ассортимента) — допродажа для Provider'ов с высоким UNDE Index. Доступ и цена привязаны к объёму Credits и Verified Actions, не к фиксированной подписке. Логика: чем активнее Provider в ядре — тем больше данных он генерирует — тем ценнее для него расширенная аналитика — тем больше он платит. Аналитика усиливает ядро, а не существует параллельно.

**Ориентир:** публичные контракты Placer.ai — $10K–31K/год [S5][S6] за аналитику посещаемости. UNDE предлагает значительно больше: не просто подсчёт трафика, а агрегированные паттерны намерений и конверсии.

## 5.3. Ускоритель 2: Enterprise (операторы пространств)

**Что:** решения для управляющих ТЦ, девелоперов, сетей — не как отдельный бизнес, а как канал дистрибуции и расширения покрытия.

- Агрегированная аналитика по всему ТЦ: потоки, мёртвые зоны, оптимальное размещение
- Сравнительные отчёты по UNDE Index: сравнение арендаторов
- Инструменты оптимизации состава арендаторов

**Модель выручки:** enterprise-контракты (Фаза 3).

## 5.4. Модель атрибуции (послойная)

Атрибуция использует Лестницу верификации (см. §4.3) — от базовой до полной верификации. Каждый следующий уровень даёт более надёжное подтверждение и больший вес в UNDE Index.

Для MVP: классы C1–C3. Для масштаба: + C4 (receipt scan). Для крупных партнёров: C5–C6 (QR / POS-интеграция).

**Ключевая механика:** клиент сканирует чек не «для Provider'а», а для прокачки своего аватара UNDE. Двойная ценность: для клиента — аватар растёт, вещь в цифровом гардеробе; для Provider'а (невидимо) — атрибуция покупки с максимальной доступной точностью без интеграции с кассой.

# 6. Local Seed — единица роста

## 6.1. Определение

*«"Всего скачало X раз" — показатель-пустышка. Единица роста UNDE — Local Seed.»*

**Local Seed** — минимальная жизнеспособная сеть в локальном кластере. Это точка, после которой экосистема начинает расти органически.

Кластер — это не обязательно молл. Это любое пространство, где люди принимают решения: торговый центр, район, lifestyle-зона, аэропорт, кампус. На старте: ТЦ.

**Local Seed = unit economics + unit of rollout.** Это не только метрика — это операционная единица, которая описывает:

- **Входные условия:** какой тип кластера, минимальное покрытие категории, каналы привлечения первых пользователей через Provider'ов
- **Пороги жизнеспособности:** при каких метриках сеть становится самоподдерживающейся (см. §6.2)
- **Unit economics:** стоимость запуска одного Seed, revenue per Seed после выхода на Фазу 2, payback period
- **Сценарий репликации:** стандартизированный процесс запуска нового Seed (набор Provider'ов → входная категория → первые пользователи → Local Seed → масштабирование)

Венчурный масштаб появляется, когда Seed реплицируется как продукт: каждый следующий запуск дешевле и быстрее предыдущего за счёт отработанного сценария.

## 6.2. Пороги Local Seed (гипотеза для ТЦ)

| Метрика | Порог | Почему |
| :---- | :---- | :---- |
| MAU в кластере | 500+ | Критическая масса для данных и виральности |
| Активные Provider'ы | 10+ | Достаточно вариантов для качественных рекомендаций |
| k-factor | > 1.0 | Сеть растёт органически |
| D30 retention | > 40% | Ритуал формируется |
| Покрытие категории | > 50% fashion-магазинов | Рекомендации полезны, а не фрагментарны |

Local Seed = все пороги выполнены одновременно.

## 6.3. Почему локальные метрики, а не глобальные

- «100K установок» может означать 1,000 юзеров в 100 кластерах = нет критической массы нигде
- Network effects работают локально: пользователь ценит UNDE в СВОЁМ молле
- Provider платит за трафик в СВОЁМ молле, не глобально

Все метрики считаются для каждого кластера отдельно:

| Метрика | Не так | А так |
| :---- | :---- | :---- |
| Пользователи | «100K установок» | «2,400 MAU в Dubai Mall» |
| Retention | «D30 = 45%» | «D30 в Dubai Mall = 52%» |
| Конверсия | «8% покупают» | «11% покупают в Dubai Mall» |
| Credits | «Всего 1M Credits» | «Zara: 12,450 Credits в Dubai Mall» |

## 6.4. Fashion + malls как плацдарм (не ограничение)

Fashion + malls — не «наша ниша», а стратегический плацдарм с лучшими условиями для доказательства тезиса:

- В закрытых ТЦ одежда и аксессуары = **57.8%** площади (без якорных арендаторов) (ICSC) [S4] — крупнейшая категория, сильная точка входа для холодного старта
- Высокая эмоциональная вовлечённость (люди любят делиться образами)
- Виральный контент (фото образов = пользовательский контент)
- Высокий средний чек и маржинальность
- Чёткая задача пользователя: «Что надеть?»
- Виртуальная примерка создаёт вау-эффект и мотивацию к шерингу

Молл: замкнутое пространство с высокой плотностью Provider'ов и покупательского трафика. Идеальная среда для доказательства Local Seed.

**Важно:** fashion доказывает, что доверие + актуальные данные + Intent Performance работают. После доказательства — те же механики (Provider / Feed / Verified Action / Credits / Index) масштабируются на любую вертикаль без переписывания модели. Fashion — плацдарм, а не потолок.

# 7. Фазы монетизации

Фазы привязаны к **порогам сети**, не к календарю. Время — ориентир, не триггер.

## 7.1. Фаза 1: Earn-only

**Триггер входа:** старт.
**Триггер выхода:** Local Seed достигнут (все пороги из §6.2).
**Ориентир:** 0–18 месяцев.

| Параметр | Значение |
| :---- | :---- |
| Credits | Только заработок, нельзя купить |
| Интеграция каталога | Бесплатно (навсегда) |
| Аналитика | Базовая (бесплатно) |
| Выручка | Нет платной монетизации. Допускаются пилотные эксперименты с willingness-to-pay (например, предложить Provider'у купить Credits в ограниченном объёме) — для валидации ценообразования, не для выручки. Условие: эксперимент не нарушает моральный контракт |
| Фокус | Максимум Local Seeds. Создание зависимости Provider'ов от данных UNDE |

На этой фазе UNDE — бесплатный инструмент, который затягивает. Provider загружает каталог, получает первые данные, зарабатывает Credits. Уходить — терять накопленное.

## 7.2. Фаза 2: Earn-to-Buy

**Триггер входа:** Local Seed достигнут в первом кластере.
**Триггер выхода:** повторяемость доказана (seed → seed → seed, 3+ кластера).
**Ориентир:** 12–36 месяцев от момента Local Seed.

| Параметр | Значение |
| :---- | :---- |
| Credits | Можно докупить, % от заработанного (коэффициент K = 0.3 → 1.0, см. §3.1) |
| Аналитика | Расширенная (бесплатно для активных Provider'ов) |
| Выручка | Credits × K × $0.10, где K — коэффициент Earn-to-Spend (гипотеза) |
| Фокус | Выручка + рост + репликация Local Seed |

## 7.3. Фаза 3: Full Stack

**Триггер входа:** повторяемый сценарий запуска доказан.
**Ориентир:** 24–48+ месяцев от момента первого Local Seed.

| Параметр | Значение |
| :---- | :---- |
| Credits | Полная механика Earn-to-Buy |
| Аналитика | Расширенная допродажа для Provider'ов с высоким Index |
| Enterprise | Решения для операторов ТЦ |
| UNDE Index | Отраслевые отчёты как B2B-продукт |
| Фокус | Масштабирование прибыли + отраслевой стандарт |

# 8. Три маховика

Венчурный масштаб UNDE — не в линейном росте, а в трёх взаимоусиливающих маховиках:

## 8.1. Маховик доверия (пользовательский)

Лучше попали в рекомендацию → пользователь доверяет больше → чаще спрашивает UNDE → больше сигналов о предпочтениях → ещё лучше попали.

Это ядро. Без маховика доверия остальные маховики не работают.

## 8.2. Маховик покрытия (данные мира)

Больше Provider'ов подключено → полнее Real-Time State Feed → точнее рекомендации → выше конверсия → конкуренты Provider'а вынуждены подключаться.

Конкурентное давление: если твой сосед по ТЦ в UNDE, а ты нет — ты невидим для людей, которые решают «что купить» через UNDE.

## 8.3. Маховик Earn-to-Spend (рост + выручка)

Provider приводит пользователей → зарабатывает Credits → тратит на приоритет → получает Verified Actions → видит окупаемость → приводит ещё больше пользователей → поддерживает свежесть данных.

Все три маховика усиливают друг друга: доверие создаёт спрос, покрытие создаёт предложение, Earn-to-Spend монетизирует и ускоряет оба.

# 9. Горизонты расширения

## 9.1. Архитектура, готовая к любой вертикали

Три абстракции (Provider / Real-Time State Feed / Verified Action) спроектированы так, чтобы Credits Economy и UNDE Index работали одинаково для любой вертикали:

| Вертикаль | Provider | Feed | Verified Action |
| :---- | :---- | :---- | :---- |
| Fashion (старт) | Магазин одежды | Наличие, размеры, цены | Визит, примерка, покупка |
| Food | Ресторан, кафе | Меню, ожидание, столики | Бронь, визит, оплата |
| Beauty | Салон, мастер | Слоты, услуги, цены | Запись, визит, оказание |
| Kids | Детский сад, секция | Расписание, места | Запись, посещение |
| Auto | Автосервис | Услуги, слоты, запчасти | Запись, визит, ремонт |

## 9.2. Два слоя ценности: глобальный аватар + локальные данные реального мира

Чтобы UNDE был «незаменимым близким», а не «локальным инструментом» — ценность должна переноситься между кластерами. Если пользователь «прокачал» аватар в Дубае, но в Москве он нулевой — это не «мой человек», это одноразовый гид.

Архитектурное решение: два слоя с разной логикой переноса.

**Слой 1: Глобальный эмоциональный актив (аватар)**

- Прокачивается от реальных действий (зал, библиотека, шопинг, встречи с друзьями)
- Хранит предпочтения: стиль, размеры, бюджет, любимые бренды, история решений
- Переносится между кластерами без потерь
- «Мой аватар ≠ твой аватар» — незаменимость. Удалить = удалить часть себя

**Слой 2: Локальный транзакционный актив (данные реального мира)**

- Уникален для каждого кластера: какие магазины, что в наличии, цены, слоты
- Обновляется через Provider'ов (Real-Time State Feed)
- Даёт ценность в день 1: «вот что доступно в зоне доступности прямо сейчас»

**Что это даёт при масштабировании:**

При входе в новый кластер пользователь приходит **с уже прокачанным аватаром**. UNDE уже знает его стиль, бюджет, размеры, историю решений. Рекомендации в новом городе сразу точнее, чем у «холодного» пользователя. 100-й кластер растёт быстрее 1-го — потому что пользователи приходят «тёплыми».

Это превращает локальную модель роста в **кумулятивную**: каждый новый Local Seed получает выгоду от всех предыдущих через портативность аватара. Линейность операционная (покрытие строится кластер за кластером), но ценность для пользователя — кумулятивная.

## 9.3. Последовательность расширения

Fashion → весь молл → lifestyle-кластеры → город.

UNDE добавляет новую категорию только когда:

1. Маховик доверия работает в текущей категории (D30 retention > порога)
2. Пользователи сами запрашивают смежные решения
3. Покрытие в новой категории достижимо в рамках существующего кластера
4. Есть Provider'ы, готовые поддерживать Real-Time State Feed

**Принцип:** UNDE — агент для решений в физическом мире (выбор → доступность → действие → подтверждение). Не «для всего на свете». Слой актуальных данных реального мира (реальное состояние мира сейчас) — это дифференциатор от универсального AI-ассистента.

## 9.4. Долгосрочное видение

UNDE = decision layer для физического мира.

**Намерение × Актуальные данные реального мира → Подтверждённое действие**

Google Maps знает *где*, но не знает *что ты хочешь*. ChatGPT знает *что ты хочешь*, но не знает *что реально доступно в зоне доступности прямо сейчас*. UNDE закрывает этот разрыв.

Глобальный офлайн-ритейл — $21.9T к 2028 (Forrester) [S9]. Никто не цифровизировал его по-настоящему. И единственный способ — не через карты или навигацию, а через ритуал и отношения.

# 10. Защитные рвы

Четыре рва по времени формирования:

## 10.1. Данные реального мира в реальном времени (операционный ров, день 1)

Данные о наличии, слотах, ценах в реальном времени. Строится кластер за кластером. Конкурент может скопировать продукт — не может скопировать покрытие.

## 10.2. Поведенческий граф (ров данных, месяц 6+)

Маршруты, предпочтения, паттерны решений конкретного пользователя. Чем дольше использует — тем точнее. Растёт экспоненциально. Конкурент, стартующий с нуля, даёт «холодные» рекомендации — UNDE уже «тёплый».

## 10.3. Ров отношений (бренд, год 1+)

Аватар, Тайная комната, The Blink, общая история — «больно терять». Мой UNDE ≠ твой UNDE: он ходил со мной в зал 47 раз, знает мою историю. Удалить UNDE = удалить часть себя. Самый сильный ров, самый медленный.

## 10.4. UNDE Index как барьер ухода (год 2+)

Накопленный индекс, Credits, репутация — барьер ухода для Provider'а. Уход из UNDE = потеря позиции, данных, статуса.

# 11. Метрики успеха

## 11.1. Главная метрика (North Star)

**Verified Actions на MAU в месяц** — сколько подтверждённых результатов генерирует средний активный пользователь.

Объединяет доверие (действует по рекомендации), покрытие (есть что рекомендовать) и монетизацию (каждое подтверждённое действие — основа выручки).

## 11.2. Опережающие индикаторы по фазам

| Фаза | Ключевые метрики |
| :---- | :---- |
| **Earn-only** | D7/D30 retention, коэффициент виральности, % покрытия категории |
| **Earn-to-Buy** | Соотношение заработанных/потраченных Credits, VA/MAU, новые Local Seeds/квартал |
| **Full Stack** | Выручка на Local Seed, принятие Index Provider'ами, валовая маржа |

## 11.3. Что НЕ считаем

| Показатель-пустышка | Почему |
| :---- | :---- |
| Общее число скачиваний | Не говорит об удержании |
| «Всего Credits выдано» | Без разбивки по кластерам бессмысленно |
| Число Provider'ов глобально | Без контекста Local Seed — раздутая цифра |

# 12. Принципы (не нарушать)

| Принцип | Что значит | Почему важно |
| :---- | :---- | :---- |
| Моральный контракт | UNDE на стороне пользователя | Ритуал важнее выручки |
| Релевантность > деньги | Буст только среди подходящих | Доверие священно |
| Органический порог | X из Y показов — без влияния Credits | Credits = приоритет, не монополия |
| Индекс нельзя купить | Только заслужить | Индекс = стандарт, не товар |
| Индекс = допуск | Без минимального Index нет доступа к Intent Performance | Рынок честный по умолчанию |
| Earn-to-Spend | Сначала вложись — потом покупай | Рост за счёт Provider'а |
| Нет комиссии с покупок | Не берём % с транзакций | UNDE ≠ маркетплейс |
| Нет продажи данных | Только агрегаты и когорты | Доверие > выручка |
| API бесплатно | Интеграция данных = $0 | Больше данных = лучше рекомендации |
| Только UNDE | Не продаём платформу под чужим брендом | Бренд = главный актив |
| Local Seed = единица | Всё по кластерам | Эффекты локальны |
| Глобальный аватар | Ценность переносится между кластерами | «Незаменимый близкий» — везде, не только в одном молле |
| Ценность без рекламы | Польза всегда | Продвижение = усиление, не замена |
